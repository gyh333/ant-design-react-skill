# i18n And Locale

Use `ConfigProvider` for Ant Design locale configuration.

```tsx
import { ConfigProvider } from 'antd'
import zhCN from 'antd/locale/zh_CN'

export const LocaleProvider = ({ children }: { children: React.ReactNode }) => (
  <ConfigProvider locale={zhCN}>{children}</ConfigProvider>
)
```

For date components, verify the project's date library configuration. Ant Design 5 commonly uses dayjs, but local projects may have additional locale setup.

Production checks:

- Confirm validation messages match the product language.
- Confirm `DatePicker`, `RangePicker`, `Calendar`, and relative dates use the expected locale.
- Keep business enums translated through the project's i18n system, not hardcoded in component files.
